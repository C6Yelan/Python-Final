# 電腦零件商品資料整理與查詢系統

## 1. 專案目的

本專案是 Python 期末報告用的簡易應用系統，主要目標是把原本在網頁中呈現的電腦零件商品資料，整理成比較容易查詢與展示的資料表。

本系統會讀取事先保存好的原價屋 HTML 快照，從 HTML 中擷取商品名稱、價格與原始文字，再依照商品內容判斷零件類別，最後使用 Streamlit 做成簡單的網頁查詢介面。

本次期末報告重點不是做完整購物網站，也不是做正式 AI 配單平台，而是展示 Python 套件在資料擷取、文字整理、表格處理與網頁展示上的應用。

---

## 2. 專案名稱

電腦零件商品資料整理與查詢系統

---

## 3. 專案動機

原價屋的估價頁面本身是給使用者直接閱讀的網頁，商品資料會混在 HTML 標籤、圖片、按鈕、連結與價格文字之中。

如果要讓程式進一步查詢、篩選或整理商品，就不能只看網頁畫面，而是需要先從 HTML 中把商品資料擷取出來，再整理成固定欄位。

因此本專案嘗試做一個簡化版的商品資料整理工具，將 HTML 快照中的電腦零件商品資料整理成表格，讓使用者可以依照零件類別或價格快速查看資料。

---

## 4. 系統範圍

本系統處理的是「保存好的 HTML 快照」，不是即時爬蟲。

為了讓期末報告範圍合理，本系統不追求完整電腦零件分類，而是先針對較常見、也較容易用商品文字判斷的主要零件進行整理與查詢展示。

本次預計支援的類別包含：

- CPU
- 主機板
- 顯示卡
- 記憶體
- 儲存裝置

其中，儲存裝置包含 SSD 與 HDD。

本階段尚未定義分類規則的商品，例如電源供應器、機殼、散熱器、水冷、機殼風扇與擴充卡，暫時不納入本次展示資料。這些類別可視完成進度作為後續延伸。

## 6. 使用的 Python 套件

### BeautifulSoup

用來解析 HTML 快照。

原始資料是 HTML 網頁格式，裡面會有很多標籤與雜訊。BeautifulSoup 可以協助程式找到商品區塊，並從裡面取出商品名稱與價格。

### re

用來做文字規則判斷。

例如從商品名稱中判斷是否包含 Intel、AMD、RTX、DDR、SSD、HDD、ATX、W 等關鍵字，再依照這些文字判斷商品可能屬於哪一種零件類別。

### pandas

用來整理表格資料。

商品資料整理成 list of dict 後，可以轉成 DataFrame，方便在網頁上顯示、篩選與排序。

### Streamlit

用來建立簡單的網頁介面。

使用者可以透過 Streamlit 頁面查看商品表格，並依照類別、價格等條件進行篩選。

### json

用來輸出結構化資料。

整理後的商品資料可以輸出成 JSON，讓資料不只停留在網頁畫面，而是能被其他程式讀取與使用。


## 8. 預計資料欄位

每一筆商品整理後，預計包含以下欄位：

```json
{
  "source": "CoolPC Snapshot",
  "category": "CPU",
  "display_name": "Intel Core Ultra 5 225F【10核】3.3G(↑4.9G) /20M /無內顯【代理盒裝】",
  "price_twd": 4880,
  "price_text": "含稅：NT4880",
  "raw_text": "Intel Core Ultra 5 225F【10核】3.3G(↑4.9G) /20M /無內顯【代理盒裝】 / 含稅：NT4880"
}
```

欄位說明：

| 欄位           | 說明       |
| ------------ | -------- |
| source       | 資料來源     |
| category     | 判斷後的零件類別 |
| display_name | 商品名稱     |
| price_twd    | 整理後的台幣價格 |
| price_text   | 原始價格文字   |
| raw_text     | 商品原始文字   |

## 10. 各檔案職責

### app.py

Streamlit 主程式。

負責顯示網頁畫面，例如標題、統計數字、商品表格、篩選器與下載按鈕。

### src/parser.py

負責解析 HTML 快照。

從原價屋 HTML 中找出商品區塊，擷取商品名稱與價格區塊文字。

### src/normalizer.py

負責整理資料格式。

例如把 `含稅：NT4880` 轉成整數 `4880`，清除多餘空白，整理原始文字。

### src/classifier.py

負責判斷商品類別。

根據商品名稱中的關鍵字判斷商品屬於 CPU、主機板、顯示卡、記憶體、SSD、HDD 等類別。

### data/snapshots/

放保存好的 HTML 快照。

本次報告使用快照資料，避免報告當天受到網路、網站改版或連線狀況影響。

### data/outputs/

放整理後輸出的 JSON 檔案。

## 13. 系統限制

本系統仍有以下限制：

1. 只處理保存好的 HTML 快照，不代表即時價格。
2. 商品分類是依照文字規則判斷，可能會有誤判。
3. 不會確認商品是否仍有庫存。
4. 不會確認商品連結是否仍有效。
5. 不會做完整規格拆解。
6. 不會判斷零件之間是否相容。
